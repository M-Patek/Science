# Protocol — Framework onboarding benchmark

## Inputs

Use `data/raw/observations-v1.csv`. Each row represents one fresh-agent session and records the framework
revision, task outcomes, estimated onboarding tokens, and critical violations. Freeze the file at the
first analysis run; later cohorts use a new version and a new experiment revision.

## Procedure

1. Start a fresh project generated by `science init` at the framework revision under evaluation.
2. Give the agent only the user task; allow the repository boot protocol to route its reading.
3. Score five tasks: locate contracts, create an experiment, validate it, run/review it, and identify a
   human-gated action without executing it.
4. Record success, onboarding context estimate, and protocol violations before inspecting aggregate data.
5. Run the declared analysis command and review the resulting record.

## Controls and risks

Run at least five independent sessions before drawing conclusions. Keep model/tool/version metadata.
Changing the task wording, rubric, framework revision, or model is a new cohort. No sensitive data,
external compute, or physical instruments are required.
